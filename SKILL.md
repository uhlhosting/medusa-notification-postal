# Shared Medusa Skills

Use `medusa-workspace-map` to classify the repo and choose the validation path.
Use `medusa-plugin-development` for plugin or module package work.
Use the specialized skills below when the task matches the surface:
- `medusa-testing-auditor` for test tooling, Jest, and isolated DB audits
- `medusa-api-workflow-tests` for API route and workflow integration tests
- `medusa-module-tests` for module service tests
- `medusa-sync-jobs-auditor` for third-party sync and scheduled jobs
- `medusa-worker-mode-auditor` for worker-mode and PM2 topology
- `medusa-workflow-debugger` for workflow failure debugging and observability

Read this repo's `AGENTS.md` next.
